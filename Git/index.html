Working directory ou working tree - todos os nossos arquivos não rastreados Stage area ou Stage tree - preparação dos nossos arquivos, para serem commitados -index Rapository (local) ou Histórico - banco de dados dos nosos arquivos - onde ficarão os pontos da história --------------------------------------------------------------------- __________________________________________________________________________ Primeira config necessária para usar o github ( git config --global user.name "your name" git config --global user.email "your email" ) Histórico do projeto no git git init = iniciar repositório ls -al .git = listar todos os arquivos do diretório .git cat .git/config cat .gitignore = ver o que tem no gitignore :q - quit :wq - write and quit git log = histórico do projeto HEAD aponta onde estamos no histórico do projeto git log --oneline = trazer histórico reduzido git log -n 5 = trazer os últimos 5 commits git log --since=2020-12-01 = trazer uma data específica git log --until=2020-12-01 = traz histórico anteriores a data específica git log --author=Emanuel = filtrar log/histórico pelo autor git log --grep=”bugfix” = traz todo log que tiver a palavra ou frase especificada git add *.html = add todos .html git add . = add tudo para o commit git diff = mostra todas as alterações do WORKING DIRECTORY versus o repositório (rastreados - aqueles que já foram commitados ao menos uma vez) git diff --staged = mostra somente o que está no stage area versus o que está no repositório git diff --color-words = mostra as palavras que foram alteradas. git rm = revome totalmente o file, não vai nem pra lixeira git restore --staged = tira o file do add para o commit git rm = para remover um file git rm -r --cached . == tirar tudo o que tinha no cache, nenhum arquivo será rastreado git add = adicionar o file, se apagar file e add novo com nome similar, git vai perceber que apenas foi renomeado git mv = mv de move, é também a mesma coisa que renomear pq muda o caminho ex: git mv repository_local.md 2-repository_local.md irá mover/renomear o file de repository_local.md para 2-repository_local.md mkdir = make directory = fazer nova pasta git restore = restaurar modificação de file antes do staged git restore --staged = restaurar file que já está no staged area git reset HEAD = também restaura file do staged area (usado antigamente) git restore --staged . = restaurar tudo - após restaurar do staged area, posso restaurar modificações de files com o git restore git commit --amend -m "msg here" = corrigir a msg do último commit feito. git checkout -- = traz um arquivo mais antigo da linha do tempo e já deixa pronto no staged area para commit git clean -n = ver files untracked que serão removidos git clean -f = força o git clean, removendo permanentemente os files untracked git revert = diretório precisa estar limpo. git revert HEAD~5 = para reverter para o HEAD -5, voltar 5 pontos na história (criará um novo commit) git log --oneline = para ver o log e copiar o hash o qual vc quer dar o revert, em seguida git revert :wq / :q git init = iniciar o git em pasta nova sem estar lincada a repositório (sem pasta .git) git commit -am "mensagem" = vai direto para o staging area, commit e cria o ponto na história(se os arquivos estão rastreados) git show = mostra as alterações feitas no inicial commit (tudo). Pode usar para ver qualquer commit git show --color-words = ver mais especifico git show -- = ver alterações dentro uma pasta, pode usar * para ver todas ex: -- src/views/* git show mostra exatamente o que tem no diretório em certo momento da história .gitignore = file adicionar arquivos que não quero no repositório.
